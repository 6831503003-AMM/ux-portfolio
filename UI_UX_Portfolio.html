01
Empathize · Pain Point Gathering

What Users Actually Struggle With

Five friction points captured from real app interactions — each analysed by what triggers the frustration, the emotional response it creates, and how severely it blocks the user's goal.

Pain Point 01
Facebook Settings & Privacy

Complex, nested settings make it hard to locate specific controls — users slow down and feel lost navigating the deep hierarchy.

😕 Confused Medium
Pain Point 02
YouTube Skip & Subscription Buttons

The skip button sits too close to ad buttons — users accidentally tap ads when in a hurry and get redirected to unwanted sites.

😤 Frustrated Medium
Pain Point 03
Messenger Notifications

Irrelevant nudges like "Someone posted a note you might like" constantly pull attention away from what the user is actually trying to do.

😤 Frustrated Low
Pain Point 04
Catholic Prayers UI

Overly mixed color palette creates visual confusion and poses an accessibility problem for users who are color-blind.

😕 Confused Medium
Pain Point 05
Shopee Pop-Up Advertisements

While using the Shopee app, many unexpected advertisements pop up. Users often accidentally click them, which redirects to annoying web pages — interrupting the shopping experience.

😠 Annoyed High

From my pain points logs, the common pattern is that these apps always try to get users' attention and interaction rather than comfort and simplicity. Consequently, their revenue-focused design decisions often create frustration when using the app. Regarding my pain point about "Facebook Settings and Privacy", if I were a developer on the team, this issue likely exists because of too many options without clear hierarchy and features overloading as the app ecosystem is very large. As a result, some users get lost in the settings and feel confused even when looking for simple adjustments.

— Research Reflection

Empathize · The 5 Whys

Root Cause Analysis

Using the 5 Whys method to trace the privacy complexity problem beyond its visible symptom — revealing that the root cause lies in how Facebook's product goals are structured, not just how settings are designed.

PROBLEM
Misplaced signals lead to complex problems.
WHY 1
Users avoid settings.
WHY 2
Menus are too deep.
WHY 3
Too many new features.
WHY 4
No UX review.
WHY 5 — ROOT CAUSE
Goals focus on quantity. (Implicitly creating UX blind spots)
01
Empathize · Causal Loop Diagram (CLD)

The Complexity Loop

A Causal Loop Diagram mapping the self-reinforcing system behind Facebook's privacy problem. Each new feature expands settings volume, which increases complexity, which drives user avoidance — and because users stop engaging with settings, the team receives no usability feedback, which justifies shipping even more features. The loop feeds itself.

Define · Point of View (POV) Statement

Framing the Problem

A Point of View statement synthesises the research into a single, actionable problem definition — keeping every subsequent design decision focused on the right user, need, and insight.

USER
Sarah Mitchell — an intermediate tech user who cares about her privacy but feels overwhelmed by complex, nested settings
+
NEED
A clear, accessible way to find and control her privacy settings quickly, without feeling confused or lost
+
INSIGHT
Facebook's continuous feature expansion creates increasingly complex menus, causing users to avoid settings altogether and unknowingly compromise their own privacy
POV STATEMENT
"Sarah Mitchell needs a simplified, transparent privacy settings experience — because Facebook's continuous feature expansion creates deeply nested menus that overwhelm her, causing her to abandon settings navigation and leave her personal data exposed by default."
02
Define · How Might We (HMW) Questions

Turning Problems into Opportunities

How Might We questions reframe each pain point as an open-ended design challenge — shifting the focus from what is broken to what could be built.

HMW 01
How might we give users a single, flat view of all their privacy controls — so that finding and adjusting settings never requires navigating nested menus?
← From: Settings complexity & deep menu navigation
HMW 02
How might we show users the current state of their privacy at a glance — so they always know what is exposed without having to actively search for it?
← From: Default settings silently compromising privacy
HMW 03
How might we let users find any privacy setting in seconds using natural language — so that not knowing where a control lives never blocks them from using it?
← From: User confusion & avoidance of settings
HMW 04
How might we introduce new features in a way that keeps the settings experience simple — so that the product can grow without making privacy harder to control?
← From: Feature expansion reinforcing the complexity loop
🔍

What the Research Revealed

  • Settings are 5+ levels deep — users abandon navigation before finding what they need
  • No visual privacy status — users have no idea what is currently exposed without digging
  • Every new feature adds a new settings entry with no consolidation or cleanup
  • Users who give up on settings leave defaults unchanged, unknowingly sharing more data
✅

How the Design Responds

  • Accounts Gateway — one flat hub, all controls visible in a single scroll
  • Traffic-light status badges show live privacy state on every setting row
  • Smart Search (⌘K) — find any setting by typing plain English, no navigation needed
  • Plain-language labels replace jargon so users understand what each toggle actually does
02
Define · User Stories

What Sarah Needs to Do

User stories translate HMW questions into concrete feature requirements — each anchored to a real user need, with acceptance criteria that make the definition of done clear and testable.

Must Have
As a Facebook user, I want to access all my privacy settings from one screen, so that I never have to dig through nested menus to find a basic control.
Acceptance Criteria
✓ Privacy hub reachable within 2 taps from the home screen
✓ All top-level privacy categories visible without scrolling
Must Have
As Sarah, I want to see a live status indicator on every privacy setting, so that I can tell at a glance whether my data is protected or exposed — without changing anything.
Acceptance Criteria
✓ Every setting row shows a colour-coded status badge (🟢 Private / 🔴 Public / 🟡 Mixed)
✓ Status updates immediately when a toggle is changed
Should Have
As an intermediate tech user, I want to type a plain-English phrase to find any setting, so that I can locate what I need even when I don't know where it lives in the menu structure.
Acceptance Criteria
✓ Search bar (⌘K) accessible from every screen in the gateway
✓ Results display setting name, category, and current status
Could Have
As a user who doesn't understand technical privacy terms, I want each setting to include a one-sentence plain-language explanation, so that I can make an informed choice without having to search for what the setting actually does.
Acceptance Criteria
✓ Info tooltip visible on tap for every privacy control
✓ Explanation is under 25 words and jargon-free

Ideate · Crazy 8s

8 Ideas in 8 Minutes

A timed Crazy 8s sprint — generating eight distinct design directions without filtering or judging, then evaluating which best addresses the root causes identified in research.

🏠
Idea 01
✓ Selected
Privacy Hub Dashboard
One top-level screen showing all privacy controls at a glance — no nested menus required.
🔍
Idea 02
✓ Selected
Smart Search Bar
Global search so users can type "who sees my posts?" and jump directly to the right setting.
🚦
Idea 03
✓ Selected
Privacy Traffic Light
Green / yellow / red status indicators on each setting so users instantly see what's exposed.
🧙
Idea 04
Guided Setup Wizard
A first-time walkthrough that lets users set their privacy preferences in three simple questions.
🃏
Idea 05
Settings Cards
Each privacy setting as a swipeable card — tap to expand details, swipe to dismiss low-priority ones.
🔔
Idea 06
Privacy Change Alerts
Proactive notification whenever a new feature silently changes a privacy default — with one-tap revert.
👤
Idea 07
Audience Previewer
Show who can see each piece of content before posting — live preview right in the composer.
📋
Idea 08
Grouped Category Tabs
Flatten settings into broad category tabs (Privacy · Security · Notifications · Data) for faster scanning.

Three ideas from the sprint rose above the rest: the Privacy Hub Dashboard (Idea 01), Smart Search (Idea 02), and Privacy Traffic Light (Idea 03). These three directly address the root causes — flat hierarchy breaks the complexity loop, search removes the need to know where things live, and status indicators make the invisible visible. They were combined into the Accounts Gateway concept for prototyping.
